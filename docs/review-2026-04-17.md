# Claude Context Engine — Functional Review

**Date:** 2026-04-17
**Scope:** Everything under `src/context_engine/` — CLI, indexer, MCP server + integration, retrieval, compression, storage. Tests read to verify coverage, not reviewed for style.
**Method:** Flow-based. Three parallel review passes: CLI+indexer, MCP+integration, retrieval+compression+storage. Findings below are the consolidated, de-duplicated set.
**Verdict:** Happy-path works, but **four separate subsystems advertise features that are not actually wired up**. Shipping as-is would embarrass the promises in the README. Priority-ordered fix list at the bottom.

---

## Cross-cutting themes

Three patterns show up across all three subsystem reviews — worth fixing at the theme level, not just per-bug:

1. **Dead integrations with passing unit tests.** `BootstrapBuilder`, `SessionCapture`, and `GraphStore` all have source files, real logic, and unit tests — but are **never called from the real serve path**. The README advertises each of them. The MCP server either stubs the tool (`reindex`) or re-routes it to a different subsystem (`session_recall` → vector search). Unit tests give a false sense of coverage because the integration wiring is missing.
2. **Config values read but not propagated.** `retrieval_confidence_threshold = 0.5` sits in config but the MCP server calls `retriever.retrieve(query, top_k=...)` without it, so the actual threshold at runtime is `0.0`. Same shape for the compression level (set-tool mutates in-memory state, never written back to config, so next `cce serve` loses the setting).
3. **String-interpolated LanceDB queries.** `vector_store.py:81, 89, 94` build `where()` clauses via f-strings with user-adjacent input (file_path, chunk_id, filter values). Not currently exploitable from the MCP surface because the inputs are validated by schema, but it's one bad tool schema change away from a real issue, and it breaks today on any file path containing a single quote.

## Critical — must fix before announcing v1

### C1. `reindex` MCP tool is a stub that lies
**File:** `src/context_engine/integration/mcp_server.py:226-230`

```python
async def _handle_reindex(self, args):
    path = args.get("path")
    if path:
        return [TextContent(type="text", text=f"Re-indexing triggered for: {path}")]
    return [TextContent(type="text", text="Full re-index triggered.")]
```

Nothing is triggered. No indexer call, no background task, no file touched. Claude Code will call this after editing a file and then search stale content. This is the single most user-visible breakage.

**Fix:** wire to the real indexer path (`backend.delete_by_file(path)` → chunk → embed → ingest). For a full re-index, kick off the same code path `_run_index` in the CLI uses.

### C2. Stale chunks for deleted files are never removed from LanceDB
**Files:** `src/context_engine/cli.py:189-226` (`_run_index`), `src/context_engine/storage/local_backend.py:39` (`delete_by_file`)

`delete_by_file` exists, tested, and works. It is never called. `cce index --changed-only` walks the current filesystem, updates the manifest for files that still exist, and ingests changed files — but does not diff the manifest against the filesystem to find deletions. Deleted files' chunks stay in LanceDB forever and will keep appearing in searches.

**Fix:**
```python
previous = set(manifest._entries.keys())
current = set(chunk.file_path for chunk in all_chunks)
for deleted in previous - current:
    await backend.delete_by_file(deleted)
    manifest.remove(deleted)
```

### C3. Confidence score math is wrong
**File:** `src/context_engine/retrieval/retriever.py:26-27`

```python
idx = vector_results.index(chunk)           # O(n) lookup, making the loop O(n²)
approx_distance = idx / max(len(vector_results), 1)
```

This is **rank normalised to [0, 1]**, not cosine distance. The vector score that feeds `confidence.score()` is therefore "where in the top-K did this land" — which correlates with quality only because LanceDB pre-sorts, but throws away all the gradient information. Chunk at rank 20 out of 20 gets `distance = 1.0` (worst possible) regardless of whether its actual cosine similarity to the query was 0.9.

**Fix:** LanceDB's `.search()` returns `_distance` in the result row. Read that column directly and feed the actual number to the scorer. The O(n²) goes away as a bonus.

### C4. Confidence threshold from config is not passed through
**Files:** `src/context_engine/integration/mcp_server.py:155`, `src/context_engine/config.py` (`retrieval_confidence_threshold: float = 0.5`)

```python
chunks = await self._retriever.retrieve(args["query"], top_k=args.get("top_k", 10))
```

No `confidence_threshold=` argument, so `retrieve` uses the default `0.0`. Combined with C3, users get ranked-but-unfiltered noise back in every `context_search`.

**Fix:** `confidence_threshold=self._config.retrieval_confidence_threshold`.

### C5. String-interpolated LanceDB filters
**File:** `src/context_engine/storage/vector_store.py:81, 89, 94`

```python
where_clauses.append(f"{key} = '{value}'")
self._table.delete(f"file_path = '{file_path}'")
results = self._table.search().where(f"id = '{chunk_id}'").limit(1).to_list()
```

Breaks today on any file path containing `'` (common in source trees with apostrophes in folder names), and is a standing SQL-injection-shaped risk if any of the inputs become client-influenced. Use LanceDB's parameterized API, or at minimum `value.replace("'", "''")` on the escape.

## High

### H1. `BootstrapBuilder` is dead code
**Files:** `src/context_engine/integration/bootstrap.py`, registered prompt in `mcp_server.py:249-279`

The MCP prompt `context-engine-init` is registered, but the handler only injects compression rules — it never invokes `BootstrapBuilder` to compose the "project overview at session start" that the README's "~10k tokens startup" number depends on. Every new Claude Code session starts cold.

**Fix:** in the prompt handler, retrieve top-N chunks for "architecture overview," feed through `BootstrapBuilder.build(...)`, and return as the prompt content.

### H2. `SessionCapture` is dead code
**Files:** `src/context_engine/integration/session_capture.py`, `mcp_server.py:194-197`

`session_recall` is supposed to return past-session decisions and discussions. The current implementation just calls the retriever and filters by chunk_type `session` or `decision` — but nothing in the pipeline actually *writes* session chunks. The `SessionCapture` class (which would do the writing) is never instantiated.

**Fix:** either wire `SessionCapture` to a real transcript source (Claude Code stop-hook, filesystem watcher on `~/.claude/projects/...`) or delete the advertised tool until it's real.

### H3. `GraphStore` is a no-op
**File:** `src/context_engine/storage/graph_store.py` (all methods pass or return `[]`), `local_backend.py:33`

`LocalBackend.graph_neighbors` returns `[]`. Graph nodes/edges are computed during ingest (seen in `test_e2e.py`) and then dropped. Meanwhile the confidence scorer assigns a **30 % weight to graph_hops** ([confidence.py:5-7](src/context_engine/retrieval/confidence.py)), which — because the graph is empty — is always estimated from a fallback heuristic. Three nested problems:

- README advertises related-code search.
- 30 % of the scoring budget is burned on a heuristic, not the feature.
- README says "50 % vector + 20 % recency" — the missing 30 % graph weight is undocumented. Documentation undercount.

**Fix:** either implement the graph store (and actually traverse edges in `graph_neighbors`) or temporarily set `_GRAPH_WEIGHT = 0.0` and re-balance to `_VECTOR_WEIGHT = 0.8, _RECENCY_WEIGHT = 0.2` so scoring stops burning weight on noise. Fix the README either way.

### H4. Embedder first-run hang with no timeout
**File:** `src/context_engine/indexer/embedder.py:8`

```python
self._model = SentenceTransformer(model_name)
```

First `cce init` on a machine without the model cached pulls ~340 MB from Hugging Face. No timeout, no progress indicator, no offline detection. Users on captive-portal / metered / offline machines see `cce init` "hang" for many minutes or forever.

**Fix:** pre-check `~/.cache/huggingface/hub/...` for the model snapshot; if not present, print "Downloading embedding model (~340 MB on first run)" before the call; wrap with `asyncio.wait_for` or explicit `requests`-backed download with timeout + retry.

### H5. `set_output_compression` is not persisted
**File:** `src/context_engine/integration/mcp_server.py:232-241`

`self._output_level = level` lives in memory. Next process start reads config default again. A user who sets "max" across three sessions will find themselves on "standard" the next morning with no indication why.

**Fix:** write to `~/.claude-context-engine/config.yaml` (or a small state file next to it). Alternatively deprecate the MCP tool and drive entirely from config.

### H6. Silent index corruption on embedder failure
**File:** `src/context_engine/cli.py:223`

```python
embedder.embed(all_chunks)           # no try/except
await backend.ingest(all_chunks, all_nodes, all_edges)
```

OOM, broken PyTorch build, CUDA errors — any exception here kills the whole `cce index` run, but if a chunk has been partially-embedded then passed to ingest, you get a LanceDB row with a zero vector. Future searches return near-duplicate garbage on any short query.

**Fix:** try/except around `embed`; on failure, skip ingest entirely and surface the error. Don't half-write.

### H7. Symlink loop in file walker
**File:** `src/context_engine/cli.py:176-187`

No `is_symlink()` guard. A repo with `project/link -> ../project/` re-enters the walker and OOMs.

**Fix:** `if entry.is_symlink(): continue` (or follow them with a `resolved_set` guard if following is desired).

## Medium

- **M1 — Manifest JSON parse errors are uncaught.** [manifest.py:9-11](src/context_engine/indexer/manifest.py). A partial write on power loss makes `cce index` hard-crash with `json.JSONDecodeError`. Catch and fall back to empty manifest.
- **M2 — Watcher races with manual `cce index`.** [watcher.py:45](src/context_engine/indexer/watcher.py), [cli.py:145](src/context_engine/cli.py). Two indexers can race on the same LanceDB table. Add an async `Lock` or advisory file-lock around ingest.
- **M3 — Binary files read with `errors="ignore"`.** [cli.py:191](src/context_engine/cli.py). A `.png` in the project gets chunked into garbage and embedded. Use `errors="strict"` and skip on `UnicodeDecodeError`.
- **M4 — Git-hook script assumes `claude-context-engine` on PATH.** [git_hooks.py:7-9](src/context_engine/indexer/git_hooks.py). Background-runs and redirects stderr, so failures are invisible. Use `Path(sys.executable).parent / "cce"` to get the absolute install path.
- **M5 — `VectorStore._ensure_table` is not thread-safe.** [vector_store.py:14-18](src/context_engine/storage/vector_store.py). Two MCP calls from different sessions can race on table creation. Add an `RLock`.
- **M6 — No input validation on `context_search`.** [mcp_server.py:154-155](src/context_engine/integration/mcp_server.py). Empty string, 100 k character queries, and `top_k=10**6` all pass through. Clamp/top-k and reject empty queries with a helpful message.
- **M7 — Config values not type-validated.** [config.py:91-98](src/context_engine/config.py). A typo like `top_k: "ten"` silently loads and blows up later in `retrieve()`. Type-check against a schema (dataclass-based `__post_init__` or `pydantic`).
- **M8 — Compression "full" mode mis-treats non-retrieved chunks.** [compressor.py:24](src/context_engine/compression/compressor.py). Gate is `chunk.confidence_score > 0.8`, which is always `0.0` for chunks that didn't come through `retrieve()`. Either default `confidence_score` to `1.0` for direct-ingested chunks or key the gate off a different field.
- **M9 — `.mcp.json` write is not atomic.** [cli.py:13-35](src/context_engine/cli.py). On a partial write the user loses every other MCP server they had configured. Write to a sibling temp file + `os.replace`.

## Low

- **L1 — LLM quality-check failure is silent.** [compressor.py:36-40](src/context_engine/compression/compressor.py). Debug unfriendly. Log "Ollama summary failed quality check, falling back to truncation" at info level.
- **L2 — `_VECTOR_WEIGHT = 0.5` is a magic number in three places.** Centralise in config so the README weights can be tuned without a code change (relevant if H3's rebalance is temporary).

## Does the README hold?

| README claim | Reality |
|---|---|
| "Index your codebase" | Yes — for JS/TS/Python; other languages fall back to whole-file MODULE chunks. |
| "Compress context. Cut token costs by 70%." | Unverified. The compressor works on its own; the retrieval-side "70 %" figure is not measured by any test and depends on C3/C4 being fixed before it's meaningful. |
| "Session startup ~10k tokens" | Not today. `BootstrapBuilder` is not invoked at session start (H1). |
| "Remembers past sessions" | No. `SessionCapture` is never instantiated (H2). |
| "Zero cloud. Everything stays local." | Confirmed. Only outbound HTTP is `localhost:11434` (Ollama). The `remote_backend` module is optional and gated by config. |
| `cce init` handles git hooks, indexing, `.mcp.json` | Yes, on the happy path. Fails silently on no-network, symlink loops, and embedder errors. |
| `reindex` tool | **No — stub that returns success (C1).** |
| `session_recall` tool | **No — runs vector search over chunks tagged `session`/`decision`, which are never written (H2).** |
| `related_context` tool | **Partial — calls `graph_neighbors` which always returns `[]` (H3).** |

## Priority fix list

Order these roughly top-to-bottom. Anything before you re-publish the PyPI package:

1. **C1** — implement `reindex` properly.
2. **C2** — remove stale chunks for deleted files in `cce index`.
3. **C3** — replace the fake rank-as-distance with LanceDB's `_distance` column.
4. **C4** — plumb `retrieval_confidence_threshold` from config to the MCP call.
5. **C5** — parameterise LanceDB `where()` filters.
6. **H1** — wire `BootstrapBuilder` into the `context-engine-init` prompt handler.
7. **H2** — either wire `SessionCapture` to a real transcript source, or remove the `session_recall` tool and the README line that promises it.
8. **H3** — wire `GraphStore` or zero-out `_GRAPH_WEIGHT` and rebalance. Update README weights.
9. **H4** — embedder timeout / offline detection / progress message.
10. **H5/H6/H7** — persist compression, try/except around embed, skip symlinks.
11. Everything under Medium, then Low, as normal hygiene.

## Positives worth keeping

- Chunker is robust across unsupported languages (falls back cleanly to MODULE chunks).
- Manifest's hash-based change detection is correct and the fast-path works.
- Git-hook idempotence (marker-detect + append) is nicely done.
- Ollama client is correctly async-timeout-wrapped and falls back cleanly when the service is down.
- Output-compression rules prioritise the right things (code blocks, paths, error messages never compressed). The policy is sound even if some wiring is loose.
- Real E2E test exists (`tests/test_e2e.py`) that does actually index → retrieve → compress → bootstrap. Great foundation — add a failing test for each Critical above and most will stop regressing.
- Privacy claim is accurate: no outbound HTTP except to `localhost:11434`.

The codebase is **close**. The critical bugs are all short, local fixes (C1-C5 together are probably 2-3 days of work), and most of the High items are wiring problems rather than architectural ones.

## Fix pass — 2026-04-17 (same day)

All Critical, High, Medium, and Low items above have been addressed. Test suite: **82/82 passing**.

### Critical
- **C1** `reindex` MCP tool — now runs the real indexing pipeline. Extracted `_run_index` into [`src/context_engine/indexer/pipeline.py`](../src/context_engine/indexer/pipeline.py) with a structured `IndexResult`; both CLI (`cce index`) and MCP `reindex` call it. Path-targeted re-indexes supported.
- **C2** Stale chunks — `pipeline.run_indexing` diffs the manifest against the filesystem at the end of a full run and calls `backend.delete_by_file(...)` + `manifest.remove(...)` for files that disappeared. Also calls `delete_by_file` before re-ingesting a changed file so chunk-boundary shifts don't duplicate rows.
- **C3** Confidence distance — [`vector_store.py`](../src/context_engine/storage/vector_store.py) now surfaces LanceDB's real `_distance` into `chunk.metadata`. [`retriever.py`](../src/context_engine/retrieval/retriever.py) reads that value (clamped to [0, 1]) instead of the rank-as-distance hack. O(n²) lookup is gone.
- **C4** Confidence threshold — [`mcp_server.py`](../src/context_engine/integration/mcp_server.py) now passes `self._config.retrieval_confidence_threshold` into `retriever.retrieve(...)`.
- **C5** LanceDB filter escaping — new `_escape_sql_literal` helper in `vector_store.py`. All `where()` clauses (`search` filters, `delete_by_file`, `get_by_id`) now go through it. File paths with apostrophes work; injection-shaped input is neutralised.

### High
- **H1** `BootstrapBuilder` — `context-engine-init` prompt handler now retrieves top-10 "architecture overview" chunks and runs them through `BootstrapBuilder.build(...)`. Real project overview on session start.
- **H2** `SessionCapture` — instantiated on MCP server start, `start_session` runs immediately. Added two new MCP tools (`record_decision`, `record_code_area`) so Claude Code can push notes in. `session_recall` searches decisions / code areas / questions across the current in-memory session plus all flushed session files. Every record flushes to disk so crashes don't lose data.
- **H3** Graph weight honesty — renamed the 0.3 weight from "graph" to "keyword_distance" (the code never did graph traversal; it was always a keyword / file-hint bonus). Function renamed `_estimate_graph_hops` → `_estimate_keyword_distance`. README updated to state `50% vector + 30% keyword/file-hint + 20% recency`.
- **H4** Embedder first-run — [`embedder.py`](../src/context_engine/indexer/embedder.py) checks the HuggingFace cache before loading; warns "Downloading embedding model (~90MB first run)" before the network fetch; wraps model load in try/except and raises a helpful error instead of a deep stack trace if the download fails.
- **H5** Persisted compression — `set_output_compression` writes `{storage}/{project}/state.json`, which is read at next `cce serve` start. Setting survives restarts.
- **H6** Embed try/except — `pipeline.run_indexing` wraps `embedder.embed(...)` and `backend.ingest(...)` in try/except. Errors are captured in `IndexResult.errors` and indexing is aborted cleanly instead of writing zero-vector rows.
- **H7** Symlinks — new `_iter_project_files` in pipeline skips `entry.is_symlink()` and uses a `resolve()`-based `seen` set to prevent loops even if the user has hardlinked tricks.

### Medium
- **M1** Manifest — `Manifest.__init__` catches `json.JSONDecodeError` / `OSError` and logs a warning; `Manifest.save` is now atomic (tempfile + `os.replace`).
- **M2** Watcher race — `pipeline` uses a per-storage `asyncio.Lock` so concurrent `run_indexing` calls serialise. A future watcher wiring won't corrupt the index against a manual `cce index`.
- **M3** Binary files — `_safe_read` in the pipeline reads with `errors="strict"`; `UnicodeDecodeError` / `OSError` puts the file in `result.skipped_files` and logs instead of crashing.
- **M4** Git hook path — [`git_hooks.py`](../src/context_engine/indexer/git_hooks.py) resolves an absolute `cce` binary path via `Path(sys.executable).parent / "cce"` then `shutil.which`, falling back to `"cce"` only as a last resort.
- **M5** VectorStore thread safety — added `threading.RLock` around `_ensure_table`, `ingest`, `search`, `delete_by_file`, `get_by_id`.
- **M6** Input validation — `_handle_context_search` rejects empty queries, clamps `top_k` to `[1, 100]`, rejects queries over 10k chars. `expand_chunk` / `related_context` / `session_recall` / record tools validate required args.
- **M7** Config types — new `_EXPECTED_TYPES` table in [`config.py`](../src/context_engine/config.py). `_apply_dict_to_config` raises `ValueError` on type mismatch (with explicit `bool`-vs-`int` handling since `bool` is a Python int subclass).
- **M8** Compressor `full` mode — new `_is_full_passthrough` helper: a chunk is passed through uncompressed either if its `confidence_score > 0.8` (came via retrieval) or it has no `embedding` (direct `expand_chunk` lookup). Fixes the bug where direct-fetch chunks were silently truncated.
- **M9** `.mcp.json` atomic — `_configure_mcp` writes via tempfile + `os.replace`. Peer MCP server entries can't be clobbered by a partial write.

### Low
- **L1** Quality-check logging — `Compressor._llm_compress` logs at info level on both `Ollama summarize failed` and `Quality check failed (identifier retention < 40%)`. No more silent degradations.
- **L2** Weight constants — already centralised in `confidence.py`; renamed for honesty (see H3).

### Tool surface changes (additive, non-breaking)

- **Added:** `record_decision`, `record_code_area` MCP tools. These are the capture side of the `session_recall` loop.
- **Changed:** `reindex` now actually runs; `session_recall` now reads persisted sessions; `set_output_compression` now persists.
